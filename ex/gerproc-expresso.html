1. Conceitos Fundamentais

O gerenciamento de processos é o coração dos Sistemas Operacionais modernos, responsável por coordenar a execução de programas e otimizar o uso dos recursos computacionais.

"A capacidade de um Sistema Operacional de gerenciar múltiplos processos simultaneamente é a base da multitarefa moderna"

Fundamentos de Sistemas Operacionais

2. Arquitetura de Processos e Threads

Processos

Unidades fundamentais de execução, definidos como instâncias de programas em execução ativa com alocações dedicadas de recursos.

Componentes
  • Segmento de texto (código)
  • Segmento de dados
  • Heap (alocação dinâmica)
  • Stack (chamadas de função)

Threads

Múltiplas linhas de execução dentro de um mesmo processo, compartilhando o mesmo espaço de endereçamento de memória.

Vantagens
  • Criação 7x mais rápida
  • Menor overhead de contexto
  • Compartilhamento de memória
  • Comunicação simplificada

3. Processos vs Threads: Comparação Detalhada

Característica Processo Thread
Compartilhamento de Recursos Não compartilha memória diretamente; cada um tem seu próprio espaço de endereço virtual Compartilha o espaço de endereço (código, dados, heap) e outros recursos
Tempo de Criação Leva mais tempo (35 µs típico) Mais rápido (5 µs típico)
Overhead de Contexto Maior sobrecarga, mudança de espaço de endereço Menor sobrecarga, mesmo espaço de endereço
Independência Processos são independentes Threads não são completamente independentes
Comunicação IPC mais complexa Comunicação simples via memória compartilhada

4. Ciclo de Vida dos Processos

O Sistema Operacional abstrai o ciclo de vida de cada processo em estados discretos, permitindo gerenciamento eficiente de recursos e multitarefa.

Novo (New)

Processo criado, PCB alocado, aguardando admissão

Pronto (Ready)

Preparado para execução, aguardando alocação de CPU

Executando

Instruções sendo executadas ativamente pela CPU

Bloqueado

Aguardando evento externo (E/S, semáforo)

Terminado

Execução concluída, recursos liberados

5. Implementação Prática

Windows: APIs Diretas

Utiliza chamadas diretas de API para gerenciamento de processos através de handles e estruturas binárias. 

HANDLE hProcess = OpenProcess(
    PROCESS_QUERY_INFORMATION | PROCESS_VM_READ,
    FALSE, processId
);
GetProcessMemoryInfo(hProcess, &pmc, sizeof(pmc));
APIs Principais:
  • EnumProcesses()
  • OpenProcess()
  • GetProcessMemoryInfo()
  • QueryFullProcessImageName()

Linux: Filesystem /proc

Segue a filosofia "tudo é arquivo", expondo informações de processos através do sistema de arquivos virtual /proc. 

std::ifstream statusFile("/proc/1234/status");
std::ifstream commFile("/proc/1234/comm");
std::ifstream cmdlineFile("/proc/1234/cmdline");
Arquivos Principais:
  • /proc/[pid]/comm
  • /proc/[pid]/status
  • /proc/[pid]/cmdline
  • /proc/[pid]/stat

6. Evolução dos Modelos de Estados

A evolução dos modelos de gerenciamento de processos reflete a necessidade de resolver limitações práticas de hardware e eficiência.

Modelo de Dois Estados

Modelo mais simples: Executando e Não-Executando. Adequado apenas para sistemas muito básicos devido à falta de distinção entre processos prontos e bloqueados.

Limitação: Agrupa processos prontos e bloqueados no mesmo estado, causando ineficiência no escalonamento.

Modelo Canônico de Cinco Estados

Resolve as deficiências do modelo anterior dividindo Não-Executando em Pronto e Bloqueado, adicionando também Novo e Terminado.

Novo → Pronto
Pronto → Executando
Executando → Bloqueado
Bloqueado → Pronto

Modelo Avançado de Sete Estados

Introduz estados suspensos para lidar com swapping e memória virtual, permitindo maior grau de multiprogramação.

Pronto/Suspenso
Processo pronto mas no armazenamento secundário
Bloqueado/Suspenso
Processo bloqueado e movido para disco

Conclusão

O gerenciamento eficaz de processos é fundamental para a multitarefa moderna, equilibrando performance, recursos e responsividade do sistema.
